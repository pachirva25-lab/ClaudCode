# Инструкция пользователя

## 1. Подготовьте папку проекта

На рабочем столе создайте папку:

```text
Выборка из Мониторинга
```

Внутри должны быть папки:

```text
Source\Мониторинг УЗРКС
Output
Logs
Skills
Documentation
Backup
Домашнее задание
```

## 2. Положите файлы мониторинга

Скопируйте файлы мониторинга в папку:

```text
Выборка из Мониторинга\Source\Мониторинг УЗРКС
```

Пример имен:

```text
2021 Мониторинг УЗРКС.xlsx
2022 Мониторинг УЗРКС.xlsx
2023 Мониторинг УЗРКС.xlsx
2024 Мониторинг УЗРКС.xlsx
2025 Мониторинг УЗРКС.xlsx
2026 Мониторинг УЗРКС.xlsx
```

## 3. Если файла нет в вашей папке

Если в проводнике Windows в папке `ClaudCode` видны только `.git`, `source-files` и `.gitkeep`, значит вы смотрите старую локальную копию без новых файлов проекта. Сначала нужно подтянуть изменения из GitHub или скачать файл отдельно. Подробный отдельный файл-инструкция: `GET_SELECTION_MONITORING_BAS.md`.

Правильное имя файла: `SelectionMonitoring.bas`. Расширение `.buzz` не используется.

На ваших скриншотах GitHub показывает только `source-files` и `.gitkeep`, а вкладка `Pull requests` пустая. Это означает, что файлов `vba`, `Documentation`, `Skills` и `SelectionMonitoring.bas` на GitHub пока нет. Сначала нужно создать/принять PR из Codex-задачи или попросить Codex создать файл в репозитории. Подробно это описано в `GET_SELECTION_MONITORING_BAS.md`.

### Где искать окно Codex / Claude Code

Codex / Claude Code — это не Проводник, не GitHub Desktop и не VS Code. Это вкладка браузера, где была открыта задача с агентом. В адресной строке обычно есть `chatgpt.com/codex/cloud/tasks/...`. На этой странице справа показывается diff/разница файлов, а вверху справа есть кнопка `Создать PR` / `Create PR`. Если вы находитесь на обычной странице GitHub `github.com/pachirva25-lab/ClaudCode`, значит вы не в том окне. Вернитесь к вкладке браузера с задачей Codex или найдите ее через историю браузера `Ctrl + H`.

## 3. Где взять файл `SelectionMonitoring.bas`

`SelectionMonitoring.bas` — это файл из данного GitHub-проекта. На вашем скриншоте открыт каталог:

```text
Рабочий стол\Cursor_VBA
```

В этой папке файла `SelectionMonitoring.bas` нет, поэтому Excel его не показывает. Нужно сначала скачать или открыть папку репозитория, где лежит файл:

```text
vba\SelectionMonitoring.bas
```

### Способ A — через GitHub Desktop

1. Откройте GitHub Desktop.
2. Выберите репозиторий `ClaudCode`.
3. Нажмите `Show in Explorer`.
4. В открывшейся папке перейдите в подпапку `vba`.
5. Там должен быть файл `SelectionMonitoring.bas`.

### Способ B — через сайт GitHub

1. Откройте репозиторий на GitHub.
2. Перейдите в папку `vba`.
3. Откройте файл `SelectionMonitoring.bas`.
4. Нажмите `Download raw file` или `Raw` → сохраните файл на компьютер.
5. Положите его, например, в папку:

```text
Рабочий стол\Выборка из Мониторинга\vba
```

## 4. Импортируйте макрос

1. Откройте файл `Выборка из Мониторинга.xlsm`.
2. Нажмите `Alt + F11`.
3. В редакторе VBA выберите `File → Import File...`.
4. В окне выбора файла перейдите не в `Cursor_VBA`, а в папку, где лежит скачанный файл:

```text
...\ClaudCode\vba
```

или:

```text
Рабочий стол\Выборка из Мониторинга\vba
```

5. Выберите файл `SelectionMonitoring.bas`. В окне импорта он может отображаться просто как `SelectionMonitoring`, потому что Windows часто скрывает расширения файлов.
6. Нажмите `Open / Открыть`.
7. Сохраните книгу в формате `.xlsm`.

## 5. Запустите макрос

1. В Excel нажмите `Alt + F8`.
2. Выберите `СформироватьВыборкуУЗРКС`.
3. Нажмите `Выполнить`.

## 6. Ответьте на вопросы макроса

### Шаг 1

Введите годы, например:

```text
2024, 2025, 2026
```

### Шаг 2

Введите варианты поиска:

```text
1, 2
```

Где:

- `1` — по номеру закупки;
- `2` — по наименованию закупки;
- `3` — по номеру ЦЗК;
- `4` — по типу сделки.

### Шаг 3

Введите значения для каждого выбранного варианта поиска.

## 7. Проверьте результат

После завершения откройте лист:

```text
Отчет УЗРКС
```

В последней колонке должна появиться колонка:

```text
Год мониторинга
```

Если возникли ошибки, откройте последний файл журнала в папке `Logs`.
