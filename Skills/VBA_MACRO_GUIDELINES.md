# Общие указания для VBA-макросов

Файл создан как общая база подходов для текущего и будущих проектов VBA.

## Совместимость

- Целевая платформа: Excel 2016 / 2021 / 2024.
- Код должен работать в 32-разрядных и 64-разрядных версиях Excel.
- По возможности избегать Windows API. Если API нужен, использовать `PtrSafe` и `LongPtr`.
- Не использовать внешние подключения, если задача решается локально.

## Структура проекта

Рекомендуемая структура локальной папки проекта:

```text
Выборка из Мониторинга/
  Source/Мониторинг УЗРКС/
  Output/
  Logs/
  Skills/
  Documentation/
  Backup/
  Домашнее задание/
```

## Подход к разработке макросов

- Включать `Option Explicit` во все модули.
- Хранить имена листов, папок и ключевых колонок в константах.
- Не завязывать перенос данных на жесткий порядок колонок, если можно использовать заголовки.
- Для больших таблиц отключать `ScreenUpdating`, `DisplayAlerts`, `EnableEvents` и обязательно возвращать настройки в блоке завершения.
- Все ошибки чтения, структуры и отсутствия файлов писать в журнал.
- Пользователю показывать понятные сообщения без технических трассировок.

## Работа с данными

- Исходные файлы открывать только для чтения.
- Результат формировать в отдельный лист/файл.
- Добавлять служебные поля, которые нужны для контроля, например `Год мониторинга`.
- Не изменять исходные книги мониторинга.

## Проверка

Минимальная проверка каждого макроса:

1. Один год и один критерий поиска.
2. Несколько годов и один критерий.
3. Один год и несколько критериев.
4. Отсутствующий файл мониторинга.
5. Отсутствующий лист-источник.
6. Пустой результат.
7. Повторный запуск макроса.
